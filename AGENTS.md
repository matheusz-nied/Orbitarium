# AGENTS.md

## Objetivo do projeto

Este projeto é uma plataforma de estudos interativos.  
Cada aula deve ser profunda, visual, didática e organizada por categoria.

O site já possui:
- página inicial
- categorias
- listagem de aulas
- página de aula
- componentes visuais
- estrutura para múltiplos temas

Ao criar uma nova aula, siga o padrão existente do projeto.

## Idioma

Todo conteúdo didático deve ser escrito em português do Brasil.

## Padrão de aula

Cada aula deve conter:

1. título
2. subtítulo
3. descrição curta
4. categoria
5. dificuldade
6. tempo estimado
7. tags
8. objetivos de aprendizagem
9. pré-requisitos
10. 8 a 14 seções didáticas
11. pelo menos 3 interações visuais
12. blocos de definição
13. blocos de insight
14. blocos de exemplo
15. blocos de erro comum
16. resumo final
17. quiz com pelo menos 8 perguntas
18. glossário com pelo menos 10 termos
19. referências confiáveis

## Pesquisa

Antes de implementar a aula, pesquise fontes confiáveis.

Priorize:
- NASA
- ESA
- CERN
- universidades
- documentação oficial
- Khan Academy
- Britannica
- Stanford Encyclopedia of Philosophy
- MIT OpenCourseWare
- papers ou materiais educacionais reconhecidos

Regras:
- não invente referências
- não copie texto literalmente
- reescreva tudo em português claro
- cite fontes na seção de referências
- não use dados numéricos específicos sem fonte confiável
- evite simplificações cientificamente erradas

## Estilo de escrita

O texto deve ser:
- claro
- profundo
- progressivo
- didático
- visual
- agradável
- não acadêmico demais
- não raso
- sem parecer texto genérico

Cada seção deve explicar:
- o problema
- a intuição
- a explicação conceitual
- exemplos
- erros comuns
- por que aquilo importa

## Design

A aula deve parecer uma experiência de estudo premium.

Use:
- bom espaçamento
- tipografia confortável
- cards claros
- blocos bem separados
- diagramas interativos
- animações suaves
- layout responsivo
- sumário navegável
- barra de progresso de leitura, se existir no projeto

Não criar uma página poluída.

## Interatividade

Cada aula deve ter pelo menos 3 interações úteis, específicas do tema.

Exemplos:
- sliders
- before/after
- timeline
- mapas interativos
- simuladores simples
- diagramas SVG
- quiz
- tooltips explicativos

As interações devem ensinar algo, não apenas decorar a página.

## Implementação

Ao adicionar uma aula:

1. siga a estrutura de dados já existente
2. adicione a aula ao catálogo
3. garanta que apareça na categoria correta
4. crie ou reutilize componentes interativos
5. mantenha consistência visual
6. rode build/lint/test disponíveis
7. corrija erros antes de finalizar

## Entrega

Ao final da tarefa, informe:
- arquivos alterados
- aula criada
- interações implementadas
- fontes usadas
- comandos executados
- pendências, se houver
